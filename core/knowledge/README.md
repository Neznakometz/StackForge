# Knowledge-паки

Библиотека готовых знаний под технологии. `/init` читает выбранные паки и
собирает из них `CLAUDE.md` проекта. Источник правил — официальные доки
технологии (низкий риск галлюцинаций), не блогпосты.

## Структура

```
knowledge/
  <tech>/
    rules.md       ← правила для CLAUDE.md (склеиваются при /init)
    SKILL.md       ← опц.: авто-триггерящийся скил под технологию
    sources.md     ← откуда взяты правила (доки, версии)
  presets/
    <name>.md      ← готовая связка стека (напр. «наш текущий стек»)
```

## Формат rules.md

```markdown
# <Технология> — правила
> Версия: <x.y> · Источник: official docs · Обновлено: YYYY-MM-DD

## Жёсткие правила
- [правило, операциональное, проверяемое]

## Path-scoped (если есть)
- `путь/**` → [что enforce]

## Конфликтует с
- [другие паки, которые нельзя ставить вместе — напр. riverpod ⊥ bloc]
```

## Правила хорошего пака

- Операциональные правила, не теория. «Используй `AsyncValue.guard`», не «пиши надёжный async».
- Помечай версию и дату — стеки протухают.
- Указывай конфликты, чтобы /init не склеил противоречия.
- Коротко: пак — это правила в CLAUDE.md, а не учебник.
