# TDD: red → green → refactor

1. Напиши тест, выражающий требование задачи. Запусти — он ДОЛЖЕН упасть (не упал → тест ничего не проверяет, перепиши).
2. Минимальный код до зелёного. Не реализуй «на будущее» (YAGNI).
3. Рефакторинг при зелёных тестах. Коммит.

Правила:
- Внешние провайдеры (API, платежи, ASR/LLM) — только моки в тестах; реальные ключи в тестах = ошибка.
- Тестируй поведение, не реализацию: завязка на приватные поля/порядок вызовов моков — хрупкость.
- Edge cases из критериев приёмки: пустой вход, нулевые значения, обрыв сети, исчерпанный лимит, таймаут.
- Без `sleep` в тестах — fake timers / явные ожидания.
- e2e — только для критических флоу фазы (перечислены в приёмке), остальное — юнитами: e2e дорог и медленен.
