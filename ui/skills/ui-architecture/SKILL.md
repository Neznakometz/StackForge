# Архитектура UI

- **Логика отдельно от представления.** Бизнес-логика — в хуках/сторах/composables, не в JSX/шаблоне. Компонент рендерит, не считает.
- **Состояния обязательны.** Каждый экран/компонент с данными реализует: loading (skeleton, не голый спиннер) · empty · error · success. Отсутствие состояния = баг ревью.
- **Компонент переиспользуем и без required-сайд-эффектов.** Пропсы типизированы; нет required-пропсов без дефолтов там, где можно.
- **Доступность с старта** (см. скил `web-a11y`), не послепродажный патч.
- **Дизайн только токенами** (см. скил `design-system`): хардкод hex/px — ошибка ревью.
- **i18n с старта:** строки через словари, не хардкод.
- **Перфоманс:** не тащи лишнее на клиент; ленивые границы, мемоизация по необходимости (не везде). Стек-специфика (RSC, useMemo, Nuxt islands) — из knowledge-пака стека.
